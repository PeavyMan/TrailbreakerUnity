Trailbreaker
When developing this game I had to make several changes and improvement
such as Adding Dash Mechanics which made the game a overall better
experience for the player.
Added Dash Mechanic
The original design included jumping as the primary mechanic. A dash
ability was added (Left Shift) to give the player more control and create
more engaging gameplay. This allows players to quickly avoid obstacles and
adds a layer of strategy.
Dash Limitation System
To balance the dash mechanic, a limit of 5 dashes per run was implemented.
A UI counter was added to display the remaining number of dashes. This
encourages players to use the ability carefully instead of relying on it
constantly.
Coin Collection System
A coin collection system was implemented to give players an additional
objective. Coins are placed throughout the level and tracked on screen.
This adds replay value and gives players a reason to take risks.
Improved Jump Mechanics
The jump system was modified from a physics-based approach to a controlled
system using manual movement. This allowed for smoother and more
consistent jumping, which better fits the endless runner style. The jump
was also adjusted to move slightly forward for better obstacle clearance.
Particle Effects
Particle effects were added to improve visual feedback. Dash actions
include particle effects, and additional effects were added to enhance
movement and make the game feel more responsive.
UI and Menu Improvements
The main menu system was improved to include:
A start button with camera transition
Fade-out effect when starting the game
On-screen UI displaying coins and remaining dashes
Character and Assets
A character model (DarkLord) was imported and used instead of a
placeholder object. This required adjustments to animations and setup,
including fixing issues such as adding a box collider so that the player's
runner stops and goes backwards when it collides with the monster.
Gameplay Adjustments
Several small adjustments were made during development:
Movement speed tuning for better control
Obstacles timing adjustments for fairness
Improved collision handling
Conclusion
While the original concept of an endless runner remained the same, the
project evolved to include additional mechanics, improved controls and
better visual feedback. These changes helped create a more polished and
engaging gameplay experience.
